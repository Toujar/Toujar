<br/>

> A full-stack enterprise workflow platform that decouples content creation from publishing — enabling secure, auditable, multi-role content pipelines without exposing creator credentials.

<div align="center">

| Attribute | Details |
|:---|:---|
| **Stack** | Spring Boot · React.js · MySQL · JWT · Spring Security · Hibernate JPA |
| **Scale** | Multi-tenant SaaS-ready architecture with role-isolated access controls |
| **Performance** | Asynchronous approval pipelines with scheduled publishing and job queuing |
| **Security** | JWT-based creator–editor session separation · No credential sharing · Audit trails |
| **Impact** | Eliminates unauthorized account access risk in creator–editor collaboration |
| **Repository** | [![GitHub](https://img.shields.io/badge/View%20on%20GitHub-6d28d9?style=flat-square&logo=github&logoColor=white)](https://github.com/Toujar) |

CreatorFlow solves a critical security and workflow problem in digital content teams: editors traditionally require access to creator accounts to upload content — exposing credentials and creating audit gaps. CreatorFlow introduces a **secure approval pipeline** where editors submit content, creators review and approve through a controlled interface, and the system handles scheduling and publishing automatically. The backend is engineered around **event-driven state transitions**, transactional integrity, and content lifecycle management — with React.js providing a clean, role-aware UI layer.

<div align="center">

| Attribute | Details |

| **Scale** | Designed for concurrent multi-user messaging with persistent session management |

| **Security** | JWT-authenticated WebSocket sessions · Protection Engine for message safeguarding |
| **Impact** | Reduces critical communication errors in high-stakes business environments |
| **Repository** | [![GitHub](https://img.shields.io/badge/View%20on%20GitHub-6d28d9?style=flat-square&logo=github&logoColor=white)](https://github.com/Toujar) |

ChatGuard Enterprise goes beyond standard messaging by introducing a **Chat Protection Engine** — a middleware layer that intercepts and flags potentially destructive user actions (bulk deletes, rapid message recalls, unauthorized forwarding) within designated protected conversations. The system delivers **typing indicators**, **read receipts**, and **connection state awareness** over a STOMP/WebSocket channel managed by a Spring Boot backend. JWT authentication is enforced at the WebSocket handshake level — ensuring that every session is cryptographically validated before message routing begins.
